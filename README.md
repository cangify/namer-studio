# Namer Studio / Ai 视频自动命名工具

Namer Studio 是一个基于 Electron 的桌面软件，用本地 Ollama 视觉模型分析视频截图，并按用户配置的命名模板批量生成视频文件名。

## 主要功能

- **批量导入视频**：支持添加单个视频、多个视频或整个文件夹。
- **自动截图分析**：软件会从视频中截取多张关键画面，发送给本地视觉模型理解内容。
- **自定义命名模板**：可配置标题01、标题02、标题03等多个片段，每个片段都有独立规则、前缀和后缀。
- **批量生成新名称**：根据视频画面和模板规则生成统一、可直接用于重命名的文件名。
- **自动重命名**：可选择只生成预览，也可开启自动重命名。
- **截图保存**：分析时截取的图片会保存到视频同目录下的 `截图/视频文件名/01.jpg` 等文件中，方便人工复核。
- **视频列表搜索和筛选**：支持按路径、文件名、新名称和状态搜索；可筛选等待处理、处理中、已生成、已重命名、失败等状态。
- **路径点击播放**：点击视频列表中的路径，可用系统默认播放器打开视频，方便确认内容。
- **远程广告位**：侧边栏广告通过线上 JSON 控制，客户端定时刷新，不需要重新打包即可更新广告图。
- **轻量更新检查**：通过线上更新 JSON 检查新版本，弹窗提示并打开浏览器下载，不做静默安装。

## 适合用来做什么

- 批量整理素材视频、短视频、AI 生成视频、剪辑素材。
- 给大量无意义文件名的视频自动生成可读标题。
- 按固定格式输出带前缀、后缀、分类字段的文件名。
- 本地离线/半离线处理敏感视频，不把视频上传到云端 AI 服务。

## Ollama 对接说明

本软件通过本机 Ollama API 调用视觉模型。默认地址是：

```text
http://127.0.0.1:11434
```

### 1. 安装 Ollama

前往 Ollama 官网下载安装：

```text
https://ollama.com
```

安装后确认 Ollama 服务已启动。

### 2. 下载视觉模型

需要使用支持图片输入的视觉模型，例如 qwen2.5vl / qwen3-vl / llava 等。示例：

```bash
ollama pull qwen2.5vl:7b
```

或使用你电脑上已经安装好的其他视觉模型。

### 3. 在软件中加载模型

1. 打开软件的“模型设置”页面。
2. 确认 Ollama 地址为 `http://127.0.0.1:11434`。
3. 点击“加载模型”。
4. 在模型下拉框里选择支持图片输入的模型。
5. 建议截图数量保持 5 张，请求超时设置为 900 秒，较大的视觉模型响应会比较慢。

### 4. 常见问题

- **加载模型失败**：确认 Ollama 已启动，地址填写正确。
- **模型没有返回内容**：当前模型可能不支持图片输入，请换视觉模型。
- **生成很慢**：视觉模型通常比较耗显卡/内存，可以减少截图数量或换更小模型。
- **命名结果不符合规则**：软件会自动重试；如果连续失败，请简化命名模板里的规则。

## 开发运行

```bash
npm install
npm start
```

## 代码检查

```bash
npm run check
```

## 打包

Windows zip：

```bash
npm run pack:win
```

Linux AppImage：

```bash
npm run pack:linux
```

macOS zip：

```bash
npm run pack:mac
```

> 注意：macOS 打包通常需要 macOS 环境才能完整生成和签名。Linux 环境下可能无法直接打出可用的 macOS 包。

## 远程配置说明

软件支持通过线上配置控制侧边栏广告和版本更新提示。相关 JSON、图片和安装包地址由运营后台维护，不写入公开 README。

## 说明

视频截图在 Electron 渲染进程里通过隐藏 video + canvas 完成，不依赖 ffmpeg。
